### **STEP 1: Get New Gemini API Key**

3. **Delete the old (leaked) key:**

   - Click the trash/delete icon

4. **Create a new API key:**
   - Click "Create API Key" button
   - Select "Create API key in new project"
   - **Copy the new key immediately!**
   - It looks like: `AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX`

### **STEP 2: Update .env File**

1. **Open this file:**
   ```
   backend/.env
   ```

2. **Replace the GEMINI_API_KEY line:**
   ```env
   PORT=5000
   MONGODB_URI=mongodb://localhost:27017/codelinkers
   JWT_SECRET=your_jwt_secret_key_change_this_in_production_12345
   JWT_EXPIRE=7d
   NODE_ENV=development
   GEMINI_API_KEY=AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
   ```
   ⚠️ Replace with YOUR new key!

3. **Save the file**

---

### **STEP 3: Verify .env is Protected**

Run these commands:

```powershell
# Check if .env is ignored
git check-ignore backend/.env
```

**Expected output:**
```
backend/.env
```
✅ Good! It's protected.

---

```powershell
# Check if .env is tracked by Git
git ls-files | Select-String "\.env"
```

**Expected output:**
```

```
✅ Good! .env is NOT in Git.

---

**If .env appears in the list above:**
```powershell

```
